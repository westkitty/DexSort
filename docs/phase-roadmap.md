# DexSort Phase Roadmap

## Phase 0 - Repository seed

Goal: establish canonical source files before app code exists.

Deliverables:

- `DexSort_Project_Bible.md`
- `README.md`
- `docs/DexSort_AI_Studio_Phases_1_to_5_Prompt.md`
- `docs/build-phase-ai-firewall.md`
- `docs/phase-roadmap.md`
- `.gitignore`

## Phase 1 - Android shell and onboarding

Build app shell, navigation, onboarding, privacy promise, settings, and placeholders.

No file mutation.

## Phase 2 - MediaStore scan and local database

Read image metadata and store it in Room.

No file mutation.

## Phase 3 - Provenance capture and device profiler

Capture original buckets/folders/source clues and show device capability mode.

No file mutation.

## Phase 4 - Favorites/manual import and exact duplicate skeleton

Support internal favorites tagging and exact duplicate grouping.

No deletion.

## Phase 5 - Multi-tag engine, concept-pack skeleton, and batch review UI

Implement tags, evidence, concept packs, Starsilk/Dexter stubs, and batch review.

No local AI required yet.

## Phase 6 - Manifest-first export preview

Only after Phases 1-5 pass: preview export plans without moving originals.

## Phase 7 - Safe copy export

Hash-verified copy-only export to user-selected destination.

## Phase 8 - Near-duplicate detection

Perceptual hash grouping and review-only near-duplicate piles.

## Phase 9 - Local AI embeddings

Only after the no-AI spine is safe: add optional local embeddings for semantic search/concept matching.

## Phase 10 - Advanced recovery and archive reports

Manifest replay, recovery reports, HTML/Markdown archive summaries, and XMP/JSON sidecars.
